# Technical Writing Portfolio 

This portfolio demonstrates how complex security workflows—such as API-driven scanning, asynchronous jobs, and error handling—can be translated into clear, developer-focused documentation.

## Focus Areas
- API documentation
- Troubleshooting and recovery guidance
- Integration workflows and real-world usage
- Docs-as-code using Markdown and GitHub

## Approach
I focus on:
- reducing user friction
- explaining systems, not just interfaces
- writing for real-world use cases and edge cases
- keeping documentation structured, concise, and actionable

## Documentation Set
- [Getting Started Guide](./api-guide/getting-started.md) – helps developers authenticate and launch their first scan with minimal setup
- [Troubleshooting Guide](./api-guide/troubleshooting.md) – helps users diagnose and resolve common API and workflow issues quickly
- [Integration Guide](./api-guide/integration-guide.md) – shows how the API can be integrated into real-world workflows and automation pipelines
- [API Reference](./api-guide/api-reference.md) – provides structured endpoint documentation, request formats, and error responses

## What This Portfolio Demonstrates
- clear API documentation structure
- developer-oriented writing
- troubleshooting and debugging mindset
- documentation designed as a scalable system rather than isolated pages
